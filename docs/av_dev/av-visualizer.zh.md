# 行动值可视化 Tab — 功能设计

## 概述

新增一个专用 Tab，用于可视化最多 4 个角色的行动值（AV）和行动顺序。核心公式为 `AV = 10000 / 速度`。用户可以直观地看到每个角色在指定 AV 范围内的行动时机，便于调整队伍速度阈值。

## 数据来源

角色速度从 `characterStore` 读取（用户导入存档后自动填充）。最终速度 = `game_data.json` 中的基础速度 + 已装备遗器的速度词条加成，与展示 Tab 中显示的数值一致，无需手动输入。

流程：选择角色 → 从 `characterStore` 读取 → 计算最终速度 → 在行动轴上标记。

## UI 布局

- 主区域为横向行动值时间轴，分多行显示
- 每行代表 100 AV
- 默认显示 3 行（0–300 AV）
- 最后一行下方有一个全宽"+"按钮，点击后追加一行（+100 AV）
- 最多选择 4 个角色，每个角色使用不同颜色区分
- 每个角色的行动标记出现在 `AV = 10000 / 速度` 处，之后以相同间隔重复，跨行连续显示

## 开发阶段

### 第一阶段 — 纯速度排轴（当前目标）
- 从角色列表中最多选择 4 个角色
- 从 characterStore 读取最终速度
- 仅根据速度在行动轴上渲染行动标记
- 不考虑任何技能、buff 或遗器效果

### 第二阶段 — 技能效果
- 模拟行动推进、加速、减速
- 需要一个简单的 AV 模拟引擎，支持在时间轴推进过程中动态修改角色 AV

### 第三阶段 — 遗器效果
- 应用有速度加成的遗器套装效果（如骇客空间等）
- 目前有速度相关效果的遗器套装数量较少

### 后续阶段（暂不在范围内）
- 技能点经济
- 能量与终结技时机
- 追加攻击
- 阶段内伤害汇总
- 敌人行动排轴与技能时机（待数据支持）

## 备注

- 模拟范围基于 AV，不按轮数计算
- 敌人相关功能等数据具备后再议
- 第一阶段可独立开发，后续阶段在此基础上叠加，不需要重构核心 UI
